Tout d’abord s’occuper de certaines choses génériques sur l’OS.

Rendre utilisateur courant membre du groupe sudo

Avec user-name votre nom d’utilisateur courant.

su
sudo adduser user-name sudo

rebooter le pc

Drivers Nvidia

Il peut arriver qu’en cas de modification récente des drivers, ceux présents sur stable ne soient pas compatibles, il faut alors aller les chercher sur unstable (Sid) dans ce cas

(à ne faire que si nécessaire. ex : démarrage en ligne de commandes si installation depuis stable).

Modifier /etc/apt/sources.list

Changer bookworm (ou votre version de Debian) par sid sur les lignes 3 et 4 :

deb http://deb.debian.org/debian/ sid main

deb-src http://deb.debian.org/debian/ sid main

Ensuite, ajoutez les éléments « contrib », « non-free » et « non-free-firmware » à la fin des deux mêmes lignes, exemple:

deb http://deb.debian.org/debian/ bookworm main contrib non-free non-free-firmware

sudo apt update
sudo apt install nvidia-driver firmware-misc-nonfree
sudo apt install nvidia-cuda-dev nvidia-cuda-toolkit
sudo apt install libnvoptix1

Repasser sid en bookworm (où votre le nom de votre version) dans le fichier sources.list si modifié.

rebooter le pc

Qgis

Installation de Qgis

article geotribu tldr;


# certificat

sudo apt update

sudo apt install ca-certificates gnupg lsb-release software-properties-common

sudo mkdir -p /etc/apt/keyrings

sudo wget -O /etc/apt/keyrings/qgis-archive-keyring.gpg \
https://download.qgis.org/downloads/qgis-archive-keyring.gpg

# ajout du dépôt

echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/qgis-archive-keyring.gpg] \
https://qgis.org/debian $(lsb_release -cs) main" | \
sudo tee /etc/apt/sources.list.d/qgis.list > /dev/null

less -F /etc/apt/sources.list.d/qgis.list

# installation

sudo apt update
sudo apt install qgis-plugin-grass

Création de l’environnement virtuel

On va ici créer un environnement virtuel avec le créateur natif de Python (venv) pour installer les packages python qui seront directement utilisables depuis Qgis (logiciel + interpréteur Python) car mamba installe certains packages d’une manière que ne sait pas lire Qgis.

Warning

Cet environnement virtuel sera à mettre à jour indépendamment et manuellement.

sudo apt install python3-pip
sudo apt install python3-venv
python3 -m venv /path/to/venv
  • Dans Qgis, ouvrir l’onglet Préférences -> Options -> Système
  • Descendre à “environnement” et ajouter “ajouter à la fin | variable : PYTHONPATH | Valeur : path/to/venv/lib/python3.11/site_packages

Pour ajouter des paquets :

  • ouvrir un terminal dans /path/to/venv/bin, puis :
./pip install nom_du_paquet

Postgres

Installation

# Create the file repository configuration:
sudo sh -c 'echo "deb https://apt.postgresql.org/pub/repos/apt \
$(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# Import the repository signing key:
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# Update the package lists:
sudo apt-get update

# Install the latest version of PostgreSQL.
# If you want a specific version, use 'postgresql-12' or similar instead of 'postgresql':
sudo apt-get -y install postgresql-16
Important

Toujours installer une version spécifique** ex : apt intall postgresql-16 pour s’éviter des problèmes de psql ou de montée de version.

Puis installation de postgis :

sudo apt install postgresql-16-postgis-3

et du Foreign Data Wrapper ogr

sudo apt install postgresql-16-ogr-fdw

Enfin, on attribue un mot de passe à l’utilisateur postgres

sudo -u postgres psql
# Une fois connecté à psql
\password
# attribuer un mot de passe
\q

PgAdmin

#
# Setup the repository
#

# Install the public key for the repository (if not done previously):
curl -fsS https://www.pgadmin.org/static/packages_pgadmin_org.pub | sudo gpg --dearmor \
-o /usr/share/keyrings/packages-pgadmin-org.gpg

# Create the repository configuration file:
sudo sh -c 'echo "deb [signed-by=/usr/share/keyrings/packages-pgadmin-org.gpg] \
https://ftp.postgresql.org/pub/pgadmin/pgadmin4/apt/$(lsb_release -cs) pgadmin4 main" \
> /etc/apt/sources.list.d/pgadmin4.list && apt update'

#
# Install pgAdmin
#

# Install for desktop mode only:
sudo apt install pgadmin4-desktop

Initialisation PostgreSQL

  1. lancer pgadmin ou via psql, sur serveurs -> register -> server (pgadmin) ou initdb / createdb (sous psql)
  2. Name : localhost | host name/adress : localhost | password : postgres_password
  3. Dans pgadmin : files -> preferences -> display -> show template databases
  4. créer un nouveau groupe utilisateur et un utilisateur associé avec les droits élargis mais non superuser
  5. Créer la base de données pour données geo, y activer postgis, postgis raster, postgis sfgcal, postgis topology, pg_stat_statements, file_fdw, postgres_fdw, ogr_fdw, tablefunc (hydra pour le stockage colonne, btree_gin pour les index multichamps mixant tabulaire et json…),
  6. se rendre sur le site pgtunes et indiquer la config désirée
  7. configurer la base avec les paramètres de pgtunes (on reste ici en local donc on peut se permettre de ne fine tuner la configuration).

::: Pour permettre à Postgres d’écouter d’autres adresses IP que 127.0.0.1, il faudra passer le paramètre listen_addresses à la valeur ’*’ en plus de modifier pg_hba.conf :::

Warning

Modifier postgresql.conf peut amener quelques surprises dans la prise en compte des paramètres. Préférer passer les paramètres avec psql (utilsateur postgres) et des ALTER SYSTEM param TO valeur;

  1. relancer le serveur postgres
sudo systemctl restart postgresql
  1. (optionnel) restaurer un dump de la base si nécessaire
  2. créer la connection depuis Qgis (avec une autentification dédiée)
